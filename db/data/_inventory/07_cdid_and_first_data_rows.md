# Phase 1.1 — 시트별 CDID 행과 첫 데이터 행 위치

본 문서는 PLAN.md Phase 1.1 일곱 번째 항목("각 시트의 통계항목 코드(CDID) 행 위치와 첫 데이터 행 번호 기록")의 산출물이다. 13회차에서 이미 본표 17개의 CDID 행이 8행에 일관 등장함을 확인했고, 본 단위는 추가로 (a) 첫 데이터 행 번호와 (b) 각 부표별 CDID 행 목록을 정리한다.

## 핵심 발견

- **모든 본표(17개) 시트의 첫 부표 CDID 행 = 8행**, **첫 데이터 행 = 9행** (예외 없음).
- 부표 간 간격은 시트별 일정:
  - Table_A·B·BX·C·D·E~K(149행 간격): 첫 부표 8행 → 두 번째 부표 157행 → 세 번째 부표 306행 → 네 번째 부표 455행 → 다섯 번째 부표 604행 → 여섯 번째 부표 753행.
  - Table_R 계열(147행 간격): 첫 부표 8행 → 두 번째 부표 155행 → 세 번째 부표 302행 → 네 번째 부표 449행 → 다섯 번째 부표 596행 → 여섯 번째 부표 743행 → 일곱 번째 부표 890행 → 여덟 번째 부표 1037행 → 아홉 번째 부표 1184행.
- 부표 갯수는 12회차 인벤토리와 100% 일치(Table_B/BX/R2 = 4, Table_C = 6, Table_R3 = 9, 기타 본표 = 3).
- 메타 시트(Cover_sheet/Notes/Records) 3개에는 CDID가 없음(분류 = meta).

## 부표 간격 패턴

| 그룹 | 부표 간격 | 비고 |
|---|---:|---|
| Table_A·B·BX·C·D·E~K (15개 본표) | 149행 | 첫 데이터 행 9 → 다음 부표 헤더 157 = 9 + 148 (시계열 행 + 빈 행) |
| Table_R 3개 (R1·R2·R3) | 147행 | 첫 데이터 행 9 → 다음 부표 헤더 155 = 9 + 146 (시계열 행 + 빈 행) |

R 계열이 2행 짧은 이유는 개정 시계열의 시작/끝 시점이 본표보다 더 짧기 때문(개정 대상 분기·연도가 한정되어 있어 시계열 길이 차이). Phase 2.1 가공 시 시트별로 부표 마지막 데이터 행을 별도 스캔해 정확한 시계열 끝을 식별해야 한다.
