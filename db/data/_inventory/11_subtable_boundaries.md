# Phase 1.1 — 부표 경계 행 인벤토리

본 문서는 PLAN.md Phase 1.1 열한 번째 항목("빈 행으로 구분되는 부표 경계 행 번호 기록")의 산출물이다. 각 본표 시트에서 부표를 분리하는 빈 행 위치를 추출해, Phase 2.1 ETL이 데이터 영역을 정확히 분할 가능하게 한다.

## 핵심 발견

- 부표 경계 빈 행 갯수 = (부표 수 − 1)이 일반 본표·R 계열 모두 일치.
- 일반 본표(Table_A·B·BX·C·D·E~K, 15개): 빈 행이 **154·303·452·601·750** 위치(필요 부표 수만큼).
- R 계열(Table_R1·R2·R3, 3개): 빈 행이 **152·299·446·593·740·887·1034·1181** 위치(필요 부표 수만큼).
- 메타 시트(Cover_sheet·Notes·Records): 빈 행이 일부 있으나 부표 경계가 아닌 가독성 분리.

## 부표별 데이터 영역 (07회차와 결합)

07회차의 CDID 행 위치 + 본 단위의 빈 행 위치를 결합하면 부표별 데이터 영역이 정확히 산출된다.

예: Table_A (3 부표)
- 부표 1: CDID r8 → 데이터 r9~r153 → 빈 행 r154 (147 데이터 행)
- 부표 2: CDID r157 → 데이터 r158~r302 → 빈 행 r303 (145 데이터 행)
- 부표 3: CDID r306 → 데이터 r307~r451 (147 데이터 행)

예: Table_R3 (9 부표)
- 부표 1: CDID r8 → 데이터 r9~r151 → 빈 행 r152 (143 데이터 행)
- ... (147행 간격) ...
- 부표 9: CDID r1184 → 데이터 r1185~r1327 (143 데이터 행)

## Phase 2.1 활용 규칙

각 시트의 부표 k개에 대해:
- `subtable_start[k] = cdid_row[k] + 1` (데이터 시작)
- `subtable_end[k] = blank_rows[k] - 1` (데이터 끝, 마지막 부표는 시트 끝까지)

이 규칙으로 부표마다 정확한 데이터 영역을 분할 → CSV 1개당 1 부표 원칙(PLAN.md Phase 2.1) 자동 충족.
