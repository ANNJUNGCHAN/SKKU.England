# db/CLAUDE.md

`db/` 디렉터리 내부 작업(원본 보관 및 데이터 가공)에 한정된 지침입니다. 저장소 전체 규칙은 루트 `CLAUDE.md`를 우선 따릅니다.

## 디렉터리 역할

- `db/source/` — 외부 기관에서 받아온 **원본 파일(읽기 전용)**. 절대 수정하지 않습니다.
- `db/data/` — `db/source/`의 원본을 가공한 결과물 저장소.

## 가공 원칙 (가장 중요)

1. **데이터 값 불변**: 셀 값, 수치, 문자열, 결측 표기를 임의로 보정·치환·반올림·재계산하지 않습니다. 단위 환산이나 합산 같은 파생 계산도 금지합니다(필요하면 별도 분석 단계에서 수행).
2. **파일 구조만 조정**: 허용되는 변경은 다음으로 제한합니다.
   - 시트 분리/병합 (값을 그대로 유지하면서 재배치)
   - 머리글 행 정리, 메타 텍스트 영역 제거
   - 열 이름 표준화 (스네이크 케이스, ASCII 권장)
   - long-form / tidy 형태로의 재구성 (pivot/unpivot)
   - 데이터 타입 명시 (날짜·기간 파싱 등 표현 형식 변경)
   - 파일 포맷 변환 (xlsx → csv/parquet 등)
3. **결측 표기 보존**: 원본의 `..`, `-`, 공란 등은 의미가 다를 수 있으므로 임의로 `NA`/`0`으로 대체하지 않습니다. 변환이 필요하면 변환 규칙을 명시한 별도 컬럼/문서에 기록합니다.

## 파일 명명

- 원본과 가공본의 대응을 추적할 수 있도록 **원본 파일명을 어간으로 유지**하고, 변환 형식을 접미사로 붙입니다.
  - 예: `balanceofpayments2025q4.xlsx` → `balanceofpayments2025q4_tidy.csv`, `balanceofpayments2025q4_<시트명>.csv`
- 시트 단위로 분리한 경우 `<원본어간>_<시트명>.<확장자>` 패턴을 사용합니다.
- 가공본 파일명은 영문 소문자·숫자·언더스코어만 사용합니다.

## 산출물에 함께 남길 정보

가공 파일을 추가할 때는 다음 항목을 README 또는 동일 디렉터리의 메모(예: `db/data/<파일명>.md`)에 기록합니다.

- 원본 파일 경로 및 시트
- 적용한 구조 변경 절차 (재현 가능하도록 순서대로)
- 컬럼별 정의·단위·기간(주기)
- 결측 표기 의미 (변경하지 않았더라도 무엇을 의미하는지 명시)

## 금지 사항

- `db/source/` 내 파일 수정·삭제·이동
- 원본에 없는 값을 가공본에 새로 만들어 넣기 (보간, 추정, 외삽)
- 출처 불명 데이터를 `db/source/` 또는 `db/data/`에 추가
