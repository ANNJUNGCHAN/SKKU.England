# 작업 계획서 (정형 CSV 가공 → 명세서 → ECOS 스타일 데이터베이스)

본 문서는 영국 거시경제 통계의 원본 데이터를 (1) 분석가가 표 형태로 즉시 호출 가능한 정형 **CSV**로 가공하고, (2) 각 변수의 의미를 한국어로 정리한 데이터 명세서를 작성하며, (3) 한국은행 ECOS의 데이터 모델을 차용한 경량 관계형 데이터베이스로 구축하기 위한 단계별 작업 계획이다.

본 계획은 **사전 분석 보고서**(영국 국제수지 2025년 4분기 자료를 시범 분석한 결과 — 시트 분류, 단위 혼재, 결측 의미, 부호 규약, 귀금속 보정, 직전 발표 대비 개정 시트 등)를 가공·명세서 설계의 도메인 근거로 삼는다. 모든 작업은 데이터 영역 내부에서 수행하며, 저장소 전반의 가공 규칙(값 불변, 구조만 조정, 결측 보존, 재현성, 한국어 문서)을 우선한다.

---

## 0. 목표와 설계 원칙

### 0.1 최종 산출물

1. **CSV 묶음** — 모든 가공 산출물 데이터는 **CSV 형식으로만 저장**한다. 엑셀(다중 시트) 형식으로 저장하지 않는다. **하나의 CSV에는 단 하나의 평면 표(원본 한 시트의 한 부표)만 담는다.** 시트 단위 가로형 CSV와, 모든 시트·계열을 단일 평면 표로 결합한 세로형 통합 CSV 두 종류를 함께 제공한다(통합 CSV도 그 자체는 단일 평면 표 구조이며, 시트·부표 정보는 컬럼으로 보존한다).
2. **데이터 명세서** — 통계표·통계항목 단위로 한국어 정의·단위·주기·결측 의미·부호 규약·근거를 정리한 문서. 사전 분석 보고서에서 도출된 도메인 맥락(부호 규약, 귀금속 보정 표와 본표의 매핑, 단위 혼재 정리, 결측 표기 분류, 개정 시트의 역할)을 함께 반영한다.
3. **ECOS 스타일 관계형 데이터베이스** — 한국은행 ECOS의 계층(통계표 → 통계항목 → 시점 → 관측치)과 표준 필드명을 차용한 경량 단일 파일 데이터베이스. 외부 서버 없이 표준 라이브러리만으로 조회·내려받기가 가능해야 한다.
4. **사용자 안내 문서** — 분석가용 호출 예시, ECOS 표준과 본 프로젝트 필드의 매핑 표, 갱신 절차.

### 0.2 ECOS 참고 구조 (반영 대상)

ECOS는 다음 4계층으로 통계를 관리한다. 본 프로젝트의 데이터 모델·필드명을 ECOS와 정렬한다.

| ECOS 계층 | 의미 | 본 프로젝트 대응 |
| --- | --- | --- |
| 통계표(STAT) | 발표 단위의 표 | 원본 엑셀의 한 시트 = 1 통계표. 한 시트에 부표가 여럿이면 부표마다 별도의 통계표로 등록 |
| 통계항목(ITEM, 최대 4단계) | 표 안의 개별 변수와 위계 | 영국 자료의 ONS CDID(예: BOKI, IKBD)를 1단계 항목 코드로 사용. 항목이 위계를 이루는 경우(예: 경상수지 → 상품무역 → 비귀금속 상품) 2~4단계 코드로 표현 |
| 주기(CYCLE)·시점(TIME) | 자료 빈도와 시점 | 주기 표기는 A(연)·S(반기)·Q(분기)·M(월)·SM(반월)·D(일) 중 사용. 시점 표기는 YYYY / YYYYSn / YYYYQn / YYYYMM / YYYYMMs / YYYYMMDD |
| 단위(UNIT_NAME)·관측치(DATA_VALUE) | 단위와 실측값 | 단위는 원문 그대로 보존(예: £ million, £ billion, % of GDP). 값은 원문 셀 문자열과 숫자 변환값 두 종을 함께 보관 |

부가 메타 필드(부모 항목 P_ITEM_CODE, 계층 레벨 LVL, 가중치 WGT, 자료 시작·종료 시점 START_TIME·END_TIME, 작성기관 ORG_NAME, 통계명 STAT_NAME, 통계표 코드 STAT_CODE, 항목명 ITEM_NAME)도 가능한 한 ECOS 필드명을 그대로 차용한다. 메타 분리는 ECOS의 통계표 목록·통계 항목 목록·통계 메타DB·통계 용어의 4축 구조를 본떠 설계한다.

### 0.3 통계표 코드 네임스페이스

ECOS는 분야별 접두 패턴(예: 100번대 통화·금융, 200번대 주요지표·국민계정, 400번대 국제수지, 700번대 물가·생산, 900번대 환율 등)으로 통계표 식별자를 부여한다. 본 프로젝트도 동일 개념을 차용해, 영국 통계는 분야별 접두를 두어 식별자(STAT_CODE)를 부여한다(예: 영국 국제수지군, 영국 GDP군, 영국 물가군 등을 분야 접두로 구분). ECOS 공식 분야 분류와 충돌하지 않는 별도 접두 규약을 두어 새 분야가 추가되어도 식별자 충돌 없이 확장되도록 한다.

### 0.4 불변 원칙

- 셀 값과 결측 표기는 절대 변경하지 않는다. 영국 국제수지 자료의 경우 사전 분석 보고서에서 확인된 'x'(비공개·미산출)와 빈 셀(계열 시작 이전 또는 자료 없음)을 **별개의 결측 코드**로 구분해 보존한다.
- 가공은 구조 재배치와 형식 변환에만 한정한다. 단위 환산·합산·보간은 금지한다. 단위 혼재(£ million / £ billion / % of GDP)는 그대로 두고, 명세서에 단위 정리 표만 별도 제공한다.
- 손으로 편집한 산출물을 두지 않으며, 모든 결과물은 동일한 절차로 재현 가능해야 한다.
- 사용자 대상 문서는 한국어로 유지한다.

## 1. 단계별 작업 계획

### Phase 1. 원본 구조 정밀 조사

- **목적**: 시트별 헤더 행 수, 부표 개수, 통계항목 코드 위치, 단위 표기, 시점 컬럼 형식, 부호 규약 진술 위치, 결측 표기 사용을 사전에 파악하여 후속 가공의 입력 사양을 확정한다.
- **시트 사전 분류**: 사전 분석 보고서를 근거로, 영국 국제수지 자료의 시트는 다음 7분류로 정리한다. (가) 메타·주석, (나) 전체 잔액 요약, (다) 경상수지(전체·귀금속 제외·EU/非EU 분해), (라) 경상수지 세부(상품무역·서비스무역·1차소득·2차소득), (마) 자본·금융계정, (바) 국제투자대조표, (사) 직전 발표 대비 개정. 신규 분야가 추가되면 이 분류 체계를 확장한다.
- **작업**: 기존 검사 도구를 확장하여 시트 단위로 다음 항목을 추출한다. (가) 시트명·분류·행·열 수, (나) 상단 메타 영역의 원문 텍스트(부호 규약·단위·발표 기간 진술 포함), (다) 통계항목 코드 행 위치와 첫 데이터 행, (라) 시점 컬럼의 형식(연·분기·월 표기 차이), (마) 단위 표기의 원문, (바) 빈 행으로 구분되는 부표 경계, (사) 사용된 결측 표기와 등장 위치. 결과는 기계 판독용 형식과 사람 검토용 형식 두 종으로 보관하되, **두 종 모두 CSV 또는 일반 텍스트 형식으로만 저장**한다.
- **종료 조건**: 모든 시트의 분류·헤더·코드·시점·단위·부표 경계·결측 표기가 한 표로 정리되어, Phase 2 가공의 입력 사양으로 그대로 사용 가능한 상태가 된다.

### Phase 2. 정형 CSV 가공

데이터 산출물은 **CSV로만 저장**하며, 엑셀(다중 시트) 형식으로 저장하지 않는다. **하나의 CSV에는 단 하나의 평면 표(한 시트의 한 부표)만 담는다.** 두 단계로 나누어 진행한다.

#### 2.1 시트 단위 가로형 CSV 분리

- **목적**: 원본의 각 시트를 한 시트당 한 CSV로 분리한다. 분석가가 원본의 모양을 그대로 받아 보고 싶을 때 사용한다.
- **분리 단위 원칙**: CSV 한 개에는 원본 시트 한 개의 한 부표만 담는다. **한 시트에 부표가 여럿이면 반드시 부표마다 별도 CSV로 분리한다.** 한 CSV 안에 두 개 이상의 시트나 부표를 결합하지 않는다.
- **작업**: 상단 메타 영역(부호 규약·단위·발표 기간 진술 포함)은 별도의 한국어 메모로 분리하여 원문을 보존하고, 본문은 정형 CSV로 추출한다. 좌측 식별자 열은 통계항목 코드라는 표준 명칭(ECOS의 ITEM_CODE에 해당)으로, 시점 열은 시점이라는 표준 명칭(ECOS의 TIME에 해당)으로 정리하며, 그 외 열 이름은 ASCII 소문자·숫자·언더스코어만 사용하도록 정리한다. 결측 표기는 원문 그대로 문자열로 보존한다.
- **검증**: 원본 셀 수와 가공 후 CSV 셀 수의 합이 일치하는지 자동으로 비교한다.

#### 2.2 세로형 통합 CSV

- **목적**: 분석에서 가장 다루기 쉬운 형태로, 모든 시트의 모든 계열을 단일 CSV(단일 평면 표)에 결합한다. 결합되어 있어도 CSV 자체는 단일 시트 구조이며, 시트·부표 정보는 컬럼으로 보존한다.
- **컬럼 사양** (열 순서 고정, ECOS 표준 필드와 정렬)

  | 열 (한국어) | ECOS 대응 | 타입 | 설명 |
  | --- | --- | --- | --- |
  | 통계표 코드 | STAT_CODE | 문자열 | 본 프로젝트 네임스페이스 규칙으로 부여한 식별자 |
  | 통계표명 | STAT_NAME | 문자열 | 한국어 통계표명 |
  | 시트 | — | 문자열 | 원본 시트 이름(부표 단위 분리 추적용) |
  | 부표 | — | 문자열 | 시트 내 부표 식별자(단일 표면 "1") |
  | 통계항목 코드 1 | ITEM_CODE1 | 문자열 | 1단계 항목 코드. 영국 자료는 ONS CDID를 우선 사용 |
  | 통계항목 코드 2 | ITEM_CODE2 | 문자열 | 2단계 항목 코드(필요 시) |
  | 통계항목 코드 3 | ITEM_CODE3 | 문자열 | 3단계 항목 코드(필요 시) |
  | 통계항목 코드 4 | ITEM_CODE4 | 문자열 | 4단계 항목 코드(필요 시) |
  | 통계항목명 | ITEM_NAME | 문자열 | 계열의 원문 라벨 |
  | 단위 | UNIT_NAME | 문자열 | 원문 단위 그대로 |
  | 주기 | CYCLE | 문자열 | A·S·Q·M·SM·D 중 하나 |
  | 시점 | TIME | 문자열 | ECOS 표기 규약 |
  | 원본 셀 | — | 문자열 | 원본 셀 문자열 그대로(결측 포함) |
  | 관측치 | DATA_VALUE | 실수 | 숫자로 파싱 가능한 경우만 채움 |

- **원칙**: 원본 셀 문자열은 항상 보존되며, 관측치(DATA_VALUE)는 편의용 보조 열이다. 결측의 의미가 분석에 중요한 경우에는 원본 셀 문자열을 우선 참조하도록 안내한다. 시점 파싱 규칙은 시트마다 다를 수 있으므로 Phase 1의 인벤토리 결과를 그대로 적용한다.
- **검증**: 통계표 코드·부표·통계항목 코드(1~4단계)·시점의 조합이 통합 CSV 안에서 유일한지 자동 점검한다.

#### 2.3 시트별 메타 메모

각 가로형 CSV 옆에 한국어 메모를 두어 (가) 원본 위치, (나) 적용한 구조 변경 절차, (다) 컬럼별 정의·단위·주기, (라) 결측 표기 의미, (마) 부호 규약, (바) 생성 절차의 요약을 기록한다. 영국 국제수지의 경우 사전 분석 보고서의 부호 규약(+ = 영국으로의 자금 유입·자산 형성·신용, − = 영국에서의 유출·부채 형성·차변. 경상수지는 + 흑자, − 적자)과 귀금속 보정 표와 본표의 관계, 개정 시트의 역할을 함께 명시한다.

### Phase 3. 데이터 명세서 작성

- **목적**: 단순한 컬럼 설명을 넘어, 각 변수가 거시경제적으로 무엇을 의미하는지를 한국어로 풀이한다. ECOS의 통계 메타DB·통계 용어 서비스에 준하는 수준의 정의가 목표다.
- **작업**: 배경지식 전용 서브에이전트에게 시트 단위로 다음을 요청한다. (가) 강의 슬라이드를 1차 근거로 한 변수의 한국어 정의, (나) 부호 규약·측정 방법·다른 변수와의 관계(예: 경상수지 = 상품 + 서비스 + 1차소득 + 2차소득, 귀금속 보정 표와 본표의 차이, 금융계정 음수의 의미가 영국으로의 순자본 유입이라는 점, 항등식 경상수지 + 자본수지 + 금융계정 + 순오차 ≡ 0), (다) 강의 자료에서 확인 불가한 항목의 명시. 강의 자료에 없는 항목은 별도로 웹 검색 전용 서브에이전트에 위임하여 출처를 보강한다.
- **산출물**: 시트·계열 단위 명세서. 컬럼은 통계표 코드(STAT_CODE)·통계표명·통계항목 코드(ITEM_CODE1~4)·원문 라벨·한국어 명칭·정의·단위(UNIT_NAME)·주기(CYCLE)·시작 시점(START_TIME)·종료 시점(END_TIME)·결측 의미·부호 규약·근거로 구성한다. 별도로 모든 통계표를 한자리에서 찾아갈 수 있는 전체 카탈로그 문서를 함께 둔다(ECOS의 통계표 목록 서비스에 대응).
- **사전 분석 보고서 활용**: 보고서에서 정리된 시트 분류·단위 혼재(£ million·£ billion·% of GDP)·결측 의미('x' vs 빈 셀)·부호 규약·귀금속 보정 표·개정 모니터링 시트 등의 도메인 맥락을 명세서의 도메인 해설에 그대로 반영한다.
- **품질 기준**: 모든 통계항목 코드 행에 한국어 정의가 채워져야 하며, 각 정의는 반드시 출처 인용을 동반한다.

### Phase 4. ECOS 스타일 관계형 데이터베이스 구축

경량 단일 파일 데이터베이스를 사용한다. 외부 서버는 두지 않으며, 분석가는 표준 라이브러리만으로 조회·내려받기가 가능해야 한다. **모든 가공 산출물 데이터는 CSV로 보관**하고, 데이터베이스는 CSV에서 적재한 조회용 사본 역할을 한다.

#### 4.1 스키마 설계 (ECOS 필드와 정렬)

테이블은 다음 다섯 종류로 구성한다.

- **통계표 메타** — 통계표 식별자(STAT_CODE, 고유 키), 통계표명(STAT_NAME, 한국어), 원문 통계표명, 분야 분류, 작성기관(ORG_NAME), 자료원, 갱신 주기, 자료 시작 시점(START_TIME), 자료 종료 시점(END_TIME), 출처 URL, 발표일, 가공 시각, 한국어 설명. 영국 국제수지군은 분야 분류에 고유 접두를 둔다.
- **통계항목 메타** — 자동 증가 식별자(고유 키), 통계표 메타에 대한 외래 키, 통계항목 코드 1~4단계(ITEM_CODE1~4), 통계항목명 1~4단계(ITEM_NAME1~4), 한국어 명칭, 한국어 정의, 부모 항목 코드(P_ITEM_CODE), 계층 레벨(LVL), 가중치(WGT), 단위(UNIT_NAME), 주기(CYCLE), 시작 시점(START_TIME), 종료 시점(END_TIME), 부호 규약. 통계표·통계항목 코드 1~4 조합에 유일성 제약을 둔다.
- **관측치** — 통계항목 메타에 대한 외래 키, 시점(TIME), 원본 셀 문자열, 관측치(DATA_VALUE). 통계항목 식별자와 시점의 조합을 기본 키로 둔다.
- **결측 사전** — 결측 코드(기본 키), 한국어 의미. 영국 국제수지의 경우 'x'(비공개·미산출), 빈 문자열(계열 시작 이전 또는 자료 없음)을 구분해 등록한다.
- **통계 용어 사전** — 용어 식별자(기본 키), 용어명(한국어·원문), 정의, 출처. ECOS의 통계 용어 서비스에 대응하며, 명세서의 부속물로 둔다.

성능을 위해 통계항목 코드 1, 시점, 그리고 통계항목·시점 복합 키에 인덱스를 둔다.

#### 4.2 적재 절차

- **입력**: Phase 2의 세로형 통합 CSV와, Phase 3의 명세서를 메타 정보로 구조화한 데이터(통계표 메타·통계항목 메타·결측 사전·용어 사전 각각을 **CSV 형태로 미리 정형화**한다. 각 메타도 한 CSV = 한 평면 표 원칙을 따른다).
- **동작**: (가) 빈 데이터베이스를 새로 생성하고 스키마를 적용한다. (나) 메타 CSV를 통계표 메타·통계항목 메타에 적재한다. (다) 세로형 통합 CSV를 관측치 테이블에 적재한다. (라) 결측 사전·용어 사전을 적재한다. (마) 마지막으로 관측치 수, 시리즈 수, 결측 비율, 시작·종료 시점 일관성 등 무결성 점검 지표를 산출한다.
- **재현성**: 적재 절차는 멱등적으로 동작해야 한다. 기존 데이터베이스 파일이 있으면 백업한 뒤 새로 생성한다.

#### 4.3 사용자 조회 인터페이스

분석가가 가장 많이 쓸 호출 방식을 ECOS Open API의 주요 서비스에 대응시켜 예시로 제공한다.

- (가) **통계항목 조회** — 통계항목 코드(ITEM_CODE1)로 시계열을 표 형태로 받아오기. ECOS의 통계 자료 검색 서비스 대응.
- (나) **통계표 목록 조회** — 분야·키워드(한국어 또는 원문)로 통계표 검색. ECOS의 통계표 목록 서비스 대응.
- (다) **통계항목 목록 조회** — 통계표 코드 단위로 항목 트리(부모-자식 관계, 레벨, 가중치 포함) 조회. ECOS의 통계 항목 목록 서비스 대응.
- (라) **선택 계열 내보내기** — 사용자가 선택한 계열을 사용자 지정 위치로 **CSV 형태**로 내보내기. 내보내기도 한 CSV = 한 평면 표 원칙을 따른다.

사용자 안내 문서에는 위 호출들의 예시 결과와 갱신 절차를 함께 기록한다.

### Phase 5. 검증·문서화·확장 준비

- **검증**: (가) 원본 셀 합계와 데이터베이스 적재 후 관측치 수가 일치하는지 자동 비교한다. (나) 무작위 표본 10건의 원본 셀 문자열을 원본 엑셀과 직접 대조한다. (다) 사전 분석 보고서에 정리된 영국 국제수지 2025년 4분기의 헤드라인 수치(상품무역 −£65.5bn, 서비스무역 +£53.3bn, 경상수지 −£18.4bn, 경상수지/GDP −2.4%, 순대외자산 −£199.8bn 등)를 데이터베이스 조회로 재계산해 일치 여부를 점검한다.
- **문서화**: 사용자 안내 문서에 데이터베이스 스키마 다이어그램, ECOS 표준과 본 프로젝트 필드 매핑 표, 호출 예시, 갱신 절차를 정리한다. 분석 보고서 문서와 데이터 인프라 안내 문서는 역할을 분리하여 운영한다.
- **확장 준비**: 새 원본 자료가 추가될 때도 Phase 1 → Phase 2 → Phase 3 → Phase 4의 절차를 동일 패턴으로 재실행할 수 있도록 절차를 일반화한다. 새 통계는 통계표 식별자에 분야 접두를 두어 분류하며(예: 영국 국제수지 계열, 영국 GDP 계열, 영국 물가 계열), ECOS 분야 분류 규약과 호환되도록 유지한다. 영국 국제수지의 경우 매 분기 발표 시 직전 발표 대비 개정 시트도 함께 적재해 개정 이력을 추적한다.

## 2. 사용 도구 및 환경

- Python 인터프리터는 저장소 루트의 가상환경 인터프리터로 고정한다. 시스템 Python이나 다른 가상환경을 사용하지 않는다.
- 추가 예정 패키지는 정형 표 입출력용 데이터 처리 패키지와, 강의 슬라이드 텍스트 추출용 패키지(필요 시)이다. 신규 패키지를 설치하면 즉시 의존성 목록을 갱신한다.
- 데이터베이스 엔진은 표준 라이브러리에 포함된 단일 파일 엔진을 사용한다.
- 서브에이전트는 다음 두 가지를 활용한다. 한국어 정의 작성에는 배경지식 전용 에이전트를 1차로 사용하고, 강의 자료에서 확인되지 않은 항목 또는 ECOS 표준 세부 사양 확인이 필요한 경우 웹 검색 전용 에이전트를 2차로 사용한다.

## 3. 다음 즉시 작업 후보

1. Phase 1을 먼저 실행하여 시트 인벤토리(분류·헤더·코드·시점·단위·부표 경계·결측 표기)를 생성한다. 값 변경이 없으므로 안전하며, 후속 단계의 입력 사양을 가장 빠르게 확정할 수 있다.
2. 또는 Phase 2.1의 가공 절차 골격을 먼저 만들고, 사전 분석 보고서가 다룬 영국 국제수지 자료의 한 시트(예: 전체 잔액 요약 시트)로 시범 적용한 뒤 결과를 검토한다.

위 두 후보 중 어느 쪽부터 시작할지 알려주면 그 단계로 진입한다.
