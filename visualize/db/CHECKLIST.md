# 체크리스트 — SQLite 시각화 부트스트랩 실행

> 본 체크리스트는 같은 폴더의 `PLAN.md` (작업 계획서) §3 작업 단계 개요 5단계와 §5 검증 기준 5건을 실행 가능한 항목으로 풀어낸 것이다. 항목은 순차 실행을 전제로 하되, 각 단계 내부 항목은 병렬로 진행해도 무방한 경우 별도 표기한다.

- [x] 시각화 전용 작업 공간(`visualize/`)을 저장소 루트에 신설한다

- [x] 본 작업의 의도·결정·실행 흐름을 담은 작업 계획서(`PLAN.md`)를 동일 위치에 1차 작성한다
- [x] 작업 계획서의 5대 미정 사항(Q1~Q5)을 사용자와 논의하여 모두 확정한다
- [x] 본 체크리스트(`CHECKLIST.md`)를 동일 위치에 작성한다

---

## 2단계 — 시각화 도구 설치

- [ ] 저장소 가상환경이 정상 활성화되는지 인터프리터 경로로 1회 확인한다
- [ ] 가상환경에 datasette 본체를 설치한다 (외부 패키지 신규 도입이므로 사용자 승인 후 진행)
- [ ] 가상환경에 Vega 기반 차트 플러그인을 함께 설치한다
- [ ] 설치 직후 의존성 명세 파일을 즉시 동기화한다 (가상환경 외부에서 재현 가능하도록)
- [ ] datasette 본체와 차트 플러그인의 버전 출력으로 정상 인식 여부를 확인한다
- [ ] datasette가 보유 SQLite 파일을 인식하는지 메타 점검 명령으로 1회 확인한다 (테이블 5건·관측치 약 7만 4천 건이 그대로 노출되는지)

---

## 3단계 — 메타데이터·캔드 쿼리 구성

- [ ] 강의 시연용 캔드 쿼리 후보 목록을 작성한다 (헤드라인 5건, 대표 CDID 시계열, 항등식 잔차 등)
- [ ] 각 캔드 쿼리의 의미·기대 결과·강의 슬라이드 매핑을 한국어로 정리한다
- [ ] 한국어 라벨·단위·출처·라이선스 표기를 메타데이터 1차 초안으로 작성한다
- [ ] 한국어 라벨의 1차 출처를 기존 사용자 가이드와 CDID 사전으로 고정하고, 임의 라벨 도입을 차단한다
- [ ] 메타데이터에 강의 보고서·강의 슬라이드의 출처 표기와 라이선스(영국 ONS·OGL v3) 안내를 포함한다
- [ ] 메타데이터·캔드 쿼리를 datasette가 읽을 수 있는 형식으로 정렬·검수한다

---

## 4단계 — 런처 구성

- [ ] 한 번의 명령으로 datasette를 기동할 수 있는 실행 래퍼를 마련한다
- [ ] 데이터베이스를 **읽기 전용(immutable)** 으로 마운트하도록 런처 옵션을 구성한다
- [ ] 메타데이터와 캔드 쿼리를 함께 적용하도록 런처 옵션을 구성한다
- [ ] 호스트 바인딩을 **로컬 루프백 한정**으로 고정하고, 외부 인터페이스에 노출되지 않도록 한다
- [ ] 포트 번호는 잘 알려진 포트를 피해 충돌 가능성을 줄이도록 지정한다
- [ ] 기동 직후 기본 브라우저가 자동으로 열리도록 구성한다
- [ ] 강의 종료 후 런처를 안전하게 종료하는 방법(터미널 종료·트레이 아이콘 등)을 안내 문구로 함께 정리한다

---

## 5단계 — 검증·시연 자료 산출

- [ ] 로컬 환경에서 웹 인터페이스(로컬 루프백 주소)에 정상 접속되는지 확인한다
- [ ] 5개 테이블이 모두 노출되며, 각 테이블의 행 수가 기존 데이터베이스 통계와 일치하는지 확인한다
- [ ] 차트 플러그인이 정상 동작하여 시계열 라인 차트, 막대 차트, 산점도가 즉석에서 그려지는지 확인한다
- [ ] 캔드 쿼리 결과가 강의 보고서의 5대 헤드라인 수치(상품 −£65.5bn / 서비스 +£53.3bn / 경상수지 −£18.4bn / 경상수지 대비 GDP −2.4% / NIIP −£199.8bn)와 정확히 일치하는지 1대1 대조한다
- [ ] 대표 시계열(예: 경상수지 분기 시계열) 한 건을 차트로 캡처한다
- [ ] 캡처본을 강의 보고서 부록 또는 시각화 영역 내부 자료로 첨부한다

---

## 검증 기준 최종 점검 (작업 완료 선언 직전)

작업 계획서 §5의 다섯 가지 기준을 한 차례씩 점검하여 모두 통과해야 작업 완료를 선언한다.

- [ ] **(검증 기준 1) 공간 분리**: 시각화 영역이 데이터·코드 영역과 명확히 분리되어 있고, 시각화 영역 단독으로 의도·범위·결정 근거를 파악할 수 있다
- [ ] **(검증 기준 2) 도구 가용성**: 가상환경에서 datasette와 차트 플러그인이 정상 인식되며, 의존성 명세에 동일 버전이 기록되어 있다
- [ ] **(검증 기준 3) 데이터 정합성**: 웹 인터페이스에 노출된 5개 테이블·관측치 수가 기존 데이터베이스의 통계와 일치한다
- [ ] **(검증 기준 4) 헤드라인 일치**: 캔드 쿼리 결과가 강의 보고서의 5대 헤드라인 수치와 정확히 일치한다
- [ ] **(검증 기준 5) 시연 가능성**: 한 번의 런처 호출만으로 강의 시연이 가능하며, 대표 시계열 차트 한 건이 캡처되어 안내 자료에 첨부되어 있다

## 위험 요인 점검 (각 단계 종료 시 1회 확인)

작업 계획서 §4의 위험 표 5건이 단계 진행 중 발현되지 않았는지 점검한다.

- [ ] 외부 패키지 신규 도입 — 의존성 명세 동기화 누락 없음, 라이선스(아파치 2.0 계열) 사전 검토 완료
- [ ] 데이터 무결성 침해 — 시각화 영역에 데이터 사본을 두지 않았고, 원본은 읽기 전용으로만 마운트되었음
- [ ] 캔드 쿼리와 보고서 수치 불일치 — 1대1 대조 검증을 통과한 캔드 쿼리만 메타데이터에 등재됨
- [ ] 한국어 라벨 누락·오역 — 모든 한국어 라벨이 기존 사용자 가이드·CDID 사전에서 유래함
- [ ] 웹서버 노출 범위 — 로컬 루프백 바인딩이 강제되어 강사 노트북 외부에서 접근 불가
