# 작업 계획서 — SQLite 시각화 부트스트랩

> 본 계획서는 보유 중인 SQLite 데이터베이스를 datasette 기반으로 웹 시각화하기 위한 사전 설계 문서입니다. 본 turn에서는 작업 공간 신설과 본 계획서 작성까지만 수행하며, 실제 도구 설치·실행은 후속 단계에서 별도 승인을 받아 진행합니다.

---

## 1. 배경 및 목적

사용자는 학부 거시경제학 강의용으로 정리된 영국 국제수지(BoP) 데이터베이스를 보유하고 있다. 해당 데이터베이스는 5개 테이블·약 7만 4천 건의 관측치로 구성된 ECOS 스타일 관계형 자료이며, 현재까지는 표준 라이브러리 기반 조회·분석만 가능한 상태였다. 이번 작업의 목적은 **별도 분석 코드 없이도 강의 참여자가 직관적으로 데이터를 탐색·시각화할 수 있는 웹 인터페이스**를 마련하는 데 있다.

사용자가 처음 언급한 "datasettle" 패키지명은 오타로 판단되며, 실제 의도는 SQLite 전용 웹 탐색·시각화 도구인 **datasette** (보조 차트 플러그인 datasette-vega 포함) 으로 해석한다. datasette는 별도의 데이터 가공 없이 기존 SQLite 파일을 그대로 마운트하여 테이블 탐색·필터·즉석 시각화·캔드 쿼리 기능을 제공하므로, 강의 자료의 정합성을 유지한 채로 인터랙티브 학습 환경을 구축하기에 적합하다.

이번 turn의 범위는 **시각화 전용 작업 공간 신설** 과 **본 작업 계획서 자체의 1차 작성** 에 한정한다. 후속 단계에서 도구 설치, 메타데이터 작성, 런처 구성, 검증을 순차적으로 진행한다.

## 2. 핵심 설계 결정

| 결정 항목 | 채택안 | 채택 사유 |
|---|---|---|
| 작업 공간 위치 | 시각화 전용 신규 영역을 별도로 두고, 기존 데이터·코드 영역과 분리한다 | 원본·가공 데이터 영역에는 데이터 무결성 규약이 강하게 적용되어 있어 시각화 부산물(메타데이터·캔드 쿼리·캡처 이미지 등)을 같은 영역에 두면 규약 충돌이 발생한다. |
| 데이터 노출 방식 | 시각화 영역에 데이터 사본을 두지 않고, 기존 데이터베이스를 읽기 전용으로 직접 참조한다 (사용자 확정) | 데이터 중복으로 인한 동기화 부담 회피 + 원본 데이터 변경 위험 차단. datasette는 immutable 모드로 마운트가 가능하므로 정책 일관성도 확보된다. **사용자가 학기 중 데이터 변경 계획이 없다고 확인**하여 회차별 사본·스냅샷도 두지 않는다. |
| 도구 선택 | datasette 본체 + Vega 기반 차트 플러그인 (사용자 확정) | SQLite 파일을 별도 ETL 없이 즉시 웹 인터페이스로 노출하는 사실상 표준 도구. 사용자 확정 의사에 따라 **datasette만 우선 도입**하며, 차트 플러그인은 함께 설치해 시계열·막대·산점 시연을 바로 가능하게 한다. 다른 시각화 도구는 본 단계에서 검토하지 않는다. |
| 노출 범위 | **강의실 로컬 시연 전용** (사용자 확정). 강사 노트북에서 로컬 호스트로만 기동하며, 학생은 강의 화면을 함께 보는 형태. | 외부·학내 네트워크 공유나 인터넷 공개를 하지 않으므로 인증·HTTPS·리버스 프록시·접근 로그 등 본격적 보안 설계 부담을 회피. 향후 필요 시 학내 공유 → 외부 공개 순으로 단계적 확장 가능. |
| 패키지 관리 | 저장소에서 이미 운영 중인 가상환경에만 설치하고, 의존성 명세 파일을 즉시 동기화한다 | 저장소 운영 규약(가상환경 고정·의존성 명세 동기화)을 그대로 따른다. |
| 본 계획서 위치 | 시각화 작업 공간 내부에 둔다 | 시각화 영역 자체를 self-documenting 하게 유지하고, 후속 작업자가 별도 검색 없이 동일 위치에서 의도·범위·결정 근거를 확인할 수 있도록 한다. |

## 3. 작업 단계 개요

### 1단계 — 작업 공간 부트스트랩 (본 turn 일부 완료)
시각화 전용 작업 공간을 신설하고, 그 안에 본 계획서를 1차 작성한다. 시각화 영역의 목적·범위·향후 단계를 사용자가 한 곳에서 파악할 수 있도록 본 계획서를 self-contained 하게 유지한다. 후속 단계에서 부속 안내 문서(폴더 README 등)는 필요에 따라 추가한다.

### 2단계 — 시각화 도구 설치
저장소 가상환경에 datasette 본체와 Vega 기반 차트 플러그인을 설치한다. 설치 직후 의존성 명세를 갱신하여 재현성을 확보하고, 도구의 정상 인식 여부를 버전 출력으로 확인한다. 본 단계는 외부 패키지 신규 설치를 동반하므로 사용자 승인을 받은 뒤 진행한다.

### 3단계 — 메타데이터·캔드 쿼리 구성
강의 시연에 활용할 캔드 쿼리(헤드라인 5건·대표 CDID 시계열·항등식 잔차 등)와 한국어 라벨·단위·출처 표기·라이선스 등을 정리한 메타데이터를 작성한다. 한국어 라벨의 1차 출처는 기존 사용자 가이드와 CDID 사전이며, 캔드 쿼리는 기존 조회 예시·헤드라인 검증 자료를 의미적으로 그대로 옮겨 적는 형태로 작성한다.

### 4단계 — 런처 구성
강의 환경에서 한 번의 명령으로 datasette를 기동할 수 있도록 간단한 실행 래퍼를 마련한다. 런처는 기존 데이터베이스를 읽기 전용으로 마운트하고, 메타데이터·캔드 쿼리를 함께 적용하며, 기본 브라우저를 자동으로 연다. **호스트 바인딩은 로컬 루프백 한정**으로 고정하여 강사 노트북 외부에서는 접근 자체가 불가능하도록 구성하며, 포트 번호도 잘 알려진 포트를 피해 충돌 가능성을 줄인다.

### 5단계 — 검증·시연 자료 산출
로컬 환경에서 웹 인터페이스에 접속해 5개 테이블이 모두 노출되는지, 차트 플러그인이 정상 동작하는지, 캔드 쿼리가 강의 보고서의 헤드라인 수치와 정확히 일치하는지 확인한다. 대표 시계열 한 건을 차트로 캡처하여 강의 보고서 부록 또는 시각화 영역 안내 문서에 시연용 자료로 첨부한다.

## 4. 위험 요인과 대응

| 위험 | 영향 | 대응 |
|---|---|---|
| 외부 패키지 신규 도입 | 의존성 명세·환경 재현성 부담 | 가상환경 고정 + 의존성 명세 즉시 동기화. 라이선스(아파치 2.0 계열) 사전 검토. |
| 데이터 무결성 침해 | 강의 자료 신뢰도 훼손 | 시각화 영역에 데이터 사본을 두지 않고, 원본을 읽기 전용으로만 마운트. 데이터 변경 권한 자체를 제거. |
| 캔드 쿼리와 보고서 수치 불일치 | 강의 시연 신뢰도 저하 | 캔드 쿼리 결과를 보고서 헤드라인과 1대1 대조하여 일치를 확인한 뒤에만 공개. |
| 한국어 라벨 누락·오역 | 학습자 혼선 | 메타데이터 라벨은 기존 사용자 가이드·CDID 사전을 단일 진실 원천으로 삼고, 신규 라벨을 임의로 도입하지 않음. |
| 웹서버 노출 범위 | 외부 접근 시 데이터 무단 사용·강의 맥락 노출 | **로컬 루프백 바인딩 고정** + 강의 시간 외 기동 금지. 학내·외부 공유 요구가 발생하면 별도 안건으로 재설계. |

## 5. 검증 기준

본 작업의 완료 여부는 다음 다섯 가지 기준으로 판정한다.

1. **공간 분리**: 시각화 영역이 데이터·코드 영역과 명확히 분리되어 있고, 시각화 영역 단독으로도 의도·범위·결정 근거를 파악할 수 있다.
2. **도구 가용성**: 가상환경에서 datasette와 차트 플러그인이 정상 인식되며, 의존성 명세에 동일 버전이 기록되어 있다.
3. **데이터 정합성**: 웹 인터페이스에 노출된 5개 테이블·관측치 수가 기존 데이터베이스의 통계와 일치한다.
4. **헤드라인 일치**: 캔드 쿼리 결과가 강의 보고서의 5대 헤드라인 수치와 정확히 일치한다.
5. **시연 가능성**: 한 번의 런처 호출만으로 강의 시연이 가능하며, 대표 시계열 차트 한 건이 캡처되어 안내 자료에 첨부되어 있다.

## 6. 사용자 확정 사항

종전 미정 사항 5건은 사용자 논의를 거쳐 모두 확정되었으며, 본 계획서는 추가 의사결정 대기 항목 없이 **실행 단계로 진입 가능**한 상태이다.

> - **(Q1)** **계획서 한 본만 유지** 로 확정. 강사용·학생용 별도 안내 문서는 두지 않으며, 후속 단계에서 필요해질 경우에만 분리 검토.
> - **(Q2)** 도구는 **datasette** 로 확정. 차트 플러그인 1종 함께 설치. 다른 시각화 도구 병행 검토는 하지 않음. (← 종전 Q4 포함하여 동시 종결)
> - **(Q3)** **데이터 직접 참조(사본 없음)** 로 확정. 학기 중 데이터 변경 계획 없음을 사용자가 확인. 회차별 스냅샷도 두지 않음.
> - **(Q5)** 노출 범위는 **강의실 로컬 시연 전용**으로 확정. 학내·외부 공개는 별도 안건으로 분리.
