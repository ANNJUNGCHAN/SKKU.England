- 강의 자료가 정식 "분야명"으로 다루는 통계는 **(a) 국제수지(BoP) + (b) 국제투자대조표(IIP)** 둘이며, IIP는 BoP의 stock 짝개념(슬라이드 25·26)으로 1:1 매핑.

- 물가·고용·생산·SNA·환율 등 여타 거시 분야명은 강의 자료 31장 어디에도 명시 없음.

### §1.2 강의 자료의 분야 위계

```
거시경제 통계
└── (강의 8주차 범위) 국제수지 분야
    ├── 제1절 국제수지표의 구성 (슬라이드 4~9)
    │   ├── 정의·특징 (슬라이드 4: flow 통계, 복식부기)
    │   └── 구성 항목 (슬라이드 5~8)
    │       ├── 경상계정 → 상품·서비스·1차소득·2차소득
    │       ├── 자본계정
    │       └── 금융계정 → 직접투자·증권투자·파생·기타·준비자산증감
    ├── 제2절 국제수지균형의 종류 (슬라이드 10~21)
    └── 제3절 국제수지와 타 사회계정과의 관계 (슬라이드 22~31)
        ├── 국민소득계정 (NIA)
        ├── 통화부문
        ├── 국제투자대조표(IIP)
        └── 결정 모형(흡수·탄력성·포트폴리오)
```

핵심: **(거시경제 통계 = 8주차 강의 영역) → 국제수지 분야 → {경상수지·자본수지·금융계정·종합수지·IIP·개정} 6개 시트군**. 이 위계는 `db/REPORT.md` 1.1절·노트 12와 1:1 정합.

## §2 ECOS 분야 분류 규약과의 정합

> **주의**: 강의 자료(`background/BoP.pptx`)에는 **ECOS 분야 분류 규약(100/200/400/700/900번대 등)이 직접 진술되지 않는다**. 본 §2는 사용자 질문의 ECOS 접두 패턴을 일반 통계 메타데이터 모델로 받아 본 프로젝트 STAT_CODE 접두와의 정합을 정성 평가한 것이며, 외부 web-search 없이 ECOS 규약 자체의 정확성을 강의 자료로 검증하지는 않았다.

### §2.1 ECOS 접두 패턴 ↔ 강의 자료 대응

| ECOS 접두 | 분야명 | 강의 자료 대응 |
|:---:|---|---|
| 100번대 | 통화·금융·주요지표 | 슬라이드 24(NFA+DC=H) 항등식 변수만 |
| 200번대 | 국민계정·국민소득 | 슬라이드 22(`Y=C+I+G+(EX−IM)`) 명시 |
| **400번대** | **국제수지(IIP 포함)** | **슬라이드 1·3·4·5·… 전 범위 — 본 강의 주력 분야** |
| 700번대 | 물가·생산 | (강의 자료 미등장) |
| 900번대 | 환율 | 슬라이드 27·28·29·31 항등식 변수만 |

### §2.2 본 프로젝트 STAT_CODE `UK_BoP_*` ↔ ECOS 400번대 정합 평가

| STAT_CODE 접두 | 본 프로젝트 의미 | ECOS 매핑 |
|---|---|---|
| `UK_BoP_Table_A` | BoP 잔액 요약 | 400 |
| `UK_BoP_Table_B/BX/C` | 경상수지 본표 | 400 |
| `UK_BoP_Table_E/F/G/H` | 경상수지 세부 | 400 |
| `UK_BoP_Table_I/J` | 자본·금융계정 | 400 |
| `UK_BoP_Table_D1_3/D4_6/D7_9/K` | IIP | 400 |
| `UK_BoP_Table_R1/R2/R3` | 국제수지 개정 | 400 (메타) |

**평가**: 본 프로젝트 STAT_CODE 접두 17개 모두 ECOS 가정 분야 400번대(국제수지)에 단일 매핑. 분야 코드 충돌 위험 없음.

### §2.3 강의 자료가 ECOS 분야 규약 자체를 다루지 않는다는 정직 명시

`background/BoP.pptx` 31장 어디에도 "ECOS"·"100번대"·"400번대"·"분야 분류 코드"·"분야 코드 체계"는 등장하지 않음. 강의 자료의 분야 위계는 IMF BPM6/한국은행 매뉴얼을 1차 근거로 삼고 있을 뿐, ECOS 분야 분류 규약과 직접 연결되지 않음(노트 02·07 §ONS특화 섹션과 정합).

### §3.3 ECOS 분야 코드 충돌 회피 STAT_CODE 접두 규약 권고

| 차원 | 본 프로젝트 표기 | ECOS 통상 표기 | 충돌 여부 |
|---|---|---|---|
| 국가·작성기관 | `UK_` 접두 (영국 ONS) | (한국은행 자료) | 비충돌 |
| 분야 코드 | `BoP` 문자 접두 | 400번대 숫자 코드 | 비충돌 (문자 vs 숫자) |
| 표 코드 | `Table_J`·`Table_D1_3` 등 ONS 표 코드 | (없음) | 비충돌 |

**권고**:
1. STAT_CODE 접두 `UK_BoP_*` 그대로 유지.
2. statcatalog.csv에 **`field_classification_main` 컬럼을 한국어("국제수지")로 두고**, **`ecos_compat_class` 컬럼에 ECOS 가까운 분야 코드("400")를 참고용**으로 병기.
3. 추후 영국 GDP·물가·환율 데이터 추가 시 `UK_GDP_*`(ECOS 200), `UK_CPI_*`(ECOS 700), `UK_FX_*`(ECOS 900) 등으로 접두 분리.

## §6 확인 못한 부분 (정직 명시)

1. **ECOS 분야 분류 규약(100/200/400/700/900번대) 자체의 정확성**: 강의 자료에 ECOS 코드 표기 없음 — 사용자 질문 진술을 그대로 받아 정성 매핑. ECOS 공식 분야 분류 코드 표·접두 규칙·세부 명칭은 한국은행 ECOS 사이트 문서·ECOS Open API 분야 트리로 별도 검증 필요(web-search 위임 권장).
2. **"국제수지" vs "외환·국제수지" 1차 분류명 선택의 ECOS 관행**: ECOS 트리에서 단독·묶음 어느 쪽인지 본 폴더 자료로 결정 불가. 강의 자료는 단독 표기만 사용 → 단독 권고.
3. **분야 분류 영문 라벨 표기 규약**: 강의 자료가 "Balance of Payments" / "B.O.P" 두 형태 혼용(슬라이드 1·25). statcatalog.csv 영문 컬럼 표준 표기는 IMF BPM6/ONS Pink Book 표기 "Balance of Payments" 단일 권고 가능 — 강의 자료 자체로는 양립.
4. **개정(Revisions)·메타 시트의 강의 자료 직접 대응**: Table_R1/R2/R3·Cover_sheet/Notes/Records는 강의 자료 미수록 → ONS 운영 규칙 분류명 사용.
5. **본 데이터셋 외 영국 거시 데이터셋 분야명**: 영국 GDP·물가·환율·고용 추가 시 분야명·STAT_CODE 접두 규약은 강의 자료 미수록 — ECOS 200/700/900번대 참고 + 외부 매뉴얼 보강 필요.
6. **강의 자료 한국어 vs 한국은행/IMF 매뉴얼 한국어 차이**: 슬라이드 5는 "본원소득수지"·"이전소득수지" 1차 표기 — ONS Pink Book 비공식 번역("1차소득"·"2차소득" 단순 표기)과의 차이는 본 폴더 자료로 결론 불가.
